# Spec Init

<background_information>
- **Mission**: Initialize a spec directory for manual spec-driven workflow. Creates `spec.json` and `requirements.md` so that `/cupola:spec-design` can be run next.
- **Success Criteria**:
  - `.cupola/specs/{spec-id}/spec.json` created with correct structure
  - `.cupola/specs/{spec-id}/requirements.md` created with project description placeholder
  - Next step guidance displayed
</background_information>

<instructions>
## Core Task
Initialize a new spec directory for **$1** by creating `spec.json` and `requirements.md`.

## Execution Steps

### Step 1: Validate Prerequisites

**Check spec-id argument**:
- If `$1` is empty or not provided, print usage and stop:
  ```
  Usage: /cupola:spec-init <spec-id> [language]
  Example: /cupola:spec-init issue-123
  Example: /cupola:spec-init issue-123 en
  ```

**Validate spec-id format**:
- spec-id must contain only alphanumeric characters, hyphens, and underscores (`[a-zA-Z0-9_-]+`)
- If invalid, stop with error:
  ```
  Error: invalid spec-id: $1
  Use alphanumeric characters, hyphens, and underscores only (e.g., issue-123, my-feature)
  ```

**Check .cupola/ directory**:
- If `.cupola/` does not exist, stop with error:
  ```
  Error: .cupola/ not found. Run: cupola init
  ```

**Check spec directory**:
- If `.cupola/specs/$1/` already exists, stop with message:
  ```
  Error: spec directory already exists: .cupola/specs/$1/
  To regenerate spec artifacts, run: /cupola:spec-design $1
  ```

### Step 2: Resolve Language

Determine the language to use (in priority order):

1. If `$2` is provided and non-empty, use it as the language
2. Otherwise, read `.cupola/cupola.toml` and extract the `language` setting if present (look for a line like `language = "ja"` — note it may be commented out with `#`)
3. If still unresolved, use `"ja"` as the default

### Step 3: Get Current Timestamp

Get the current UTC timestamp in ISO8601 format: `YYYY-MM-DDTHH:MM:SSZ`

Use Bash: `date -u +"%Y-%m-%dT%H:%M:%SZ"`

### Step 4: Generate spec.json

**Try to read template** from `.cupola/settings/templates/specs/init.json`:
- If the template exists, read it and replace:
  - `{{FEATURE_NAME}}` → `$1`
  - `{{TIMESTAMP}}` → current UTC timestamp (same value for both `created_at` and `updated_at`)
  - `{{LANGUAGE}}` → resolved language code
- If the template does not exist, use this fallback JSON (replace values inline):
  ```json
  {"feature_name":"$1","created_at":"<timestamp>","updated_at":"<timestamp>","language":"<language>","phase":"initialized","approvals":{"requirements":{"generated":false,"approved":false},"design":{"generated":false,"approved":false},"tasks":{"generated":false,"approved":false}},"ready_for_implementation":false}
  ```

**Create directory and write file**:
- Create `.cupola/specs/$1/` directory
- Write the generated content to `.cupola/specs/$1/spec.json`

### Step 5: Generate requirements.md

**Try to read template** from `.cupola/settings/templates/specs/requirements-init.md`:
- If the template exists, read it and replace:
  - `{{PROJECT_DESCRIPTION}}` → `(ここにプロジェクトの説明を記入してください)` (or appropriate placeholder in the resolved language)
- If the template does not exist, use this fallback content:
  ```markdown
  # Requirements Document

  ## Project Description (Input)
  (ここにプロジェクトの説明を記入してください)

  ## Requirements
  <!-- Will be generated by /cupola:spec-design -->
  ```

**Write file**:
- Write the generated content to `.cupola/specs/$1/requirements.md`

### Step 6: Display Completion Message

Print a success summary:

```
Spec directory initialized: .cupola/specs/$1/

Files created:
  - .cupola/specs/$1/spec.json
  - .cupola/specs/$1/requirements.md

Next steps:
  1. Edit .cupola/specs/$1/requirements.md
     Add your project description in the "Project Description (Input)" section
  2. Run: /cupola:spec-design $1
     Generates requirements, design, and tasks
```

## Critical Constraints
- **Idempotent check**: Never overwrite an existing spec directory
- **Template fallback**: Always generate files even when templates are missing
- **Fail fast**: Validate all preconditions before creating any files
- **No Rust daemon interaction**: This skill operates purely on the file system
</instructions>

## Tool Guidance
- **Bash**: Use for timestamp generation and directory existence checks
- **Read**: Load template files and cupola.toml
- **Write**: Create spec.json and requirements.md

## Output Description

Provide a brief summary:

1. **Files Created**: List of created file paths
2. **Language**: Resolved language code and its source (argument / cupola.toml / default)
3. **Next Step**: `/cupola:spec-design $1` with instruction to fill in project description first

**Format**: Concise (under 100 words)

## Safety & Fallback

### Error Scenarios
- **Missing spec-id**: Print usage and stop — do not create any files
- **Invalid spec-id characters**: Report error and stop — do not create any files
- **Missing .cupola/**: Instruct user to run `cupola init` first
- **Existing spec directory**: Report the path and suggest `/cupola:spec-design` — do not overwrite
- **Missing templates**: Use inline fallback values and continue normally
