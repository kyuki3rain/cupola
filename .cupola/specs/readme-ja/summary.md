# readme-ja サマリ

## Feature
英語版 `README.md` の日本語版 `README.ja.md` をリポジトリルートに作成し、両者間に言語切り替えリンクを設置。日本語話者の開発者・利用者がプロジェクト概要・セットアップ・使い方を母語で参照できるようにする。

## 要件サマリ
- `README.ja.md` を新規作成し、README.md 全 8 セクション（Project Overview / Prerequisites / Installation & Setup / Usage / CLI Command Reference / Configuration Reference / Architecture Overview / License）に 1:1 対応する日本語セクションを持つ。
- 見出しレベル・セクション順序を原文と同一に維持、コードブロック・コマンド例・テーブル内の技術項目（ツール名、オプション名、設定キー名）は原文維持。
- 自然な日本語文体（機械翻訳調を避ける）。
- 双方のタイトル直下に相互リンク（`[日本語](./README.ja.md)` / `[English](./README.md)`）をインライン配置。
- 目次のアンカーリンクは日本語見出しに対応。

## アーキテクチャ決定
- **タイトル直下にインラインリンク配置** (採用): OSS プロジェクトで広く採用されている最も視認性の高いパターン。独立セクション化は 3 言語以上への拡張時の将来課題とする。
- **技術用語の選択的翻訳**: コードブロック・コマンド・設定キー・ライブラリ名は原文維持、説明文中の一般的技術用語（Build, Test 等）は文脈に応じて日本語化。正確性と可読性の両立を重視。
- **相対パス `./README.md` / `./README.ja.md` を使用**: GitHub の Markdown レンダリングで安定動作。
- **日本語アンカーリンクは GitHub 自動生成に準拠**: 例 `## プロジェクト概要` → `#プロジェクト概要`。
- **翻訳同期の自動化は非スコープ**: CONTRIBUTING ガイド等での運用ルール化に委ねる。

## コンポーネント
- `README.ja.md`（新規）: 日本語版ドキュメント本体。
- `README.md`: タイトル `# Cupola` 直下に `[日本語](./README.ja.md)` リンクを 1 行追加するのみ。他の既存内容は変更しない。

## 主要インターフェース
ドキュメント追加のみ。コード変更・ビルドへの影響なし。検証は手動の目視＋リンク遷移確認で実施。

## 学び / トレードオフ
- README.md 更新時に README.ja.md が追従しないリスクは残るが、本スコープ外としてルール化に委ねる判断。
- セクション見出しを日本語化することで TOC の一貫性が得られる一方、英語でのページ内検索性は若干低下する。
- 技術項目原文維持は翻訳判断が主観的になる箇所があるが、「コマンド・設定キーは必ず原文、説明列は翻訳」という明確なルールで判断コストを抑えた。
