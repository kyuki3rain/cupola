# Implementation Plan

- [x] 1. `/cupola:fix` スキルのコミットメッセージ改善
- [x] 1.1 design/impl の区別に応じた適切なコミットプレフィックスと動的メッセージ生成を実現する
  - `design` 指定時は `docs:` プレフィックス、`impl` 指定時は `fix:` プレフィックスを付与する
  - 固定文字列のコミットメッセージ本文を廃止し、変更内容に基づいて動的にメッセージを生成するよう指示する
  - コミット例を design/impl それぞれ別々に示し、角括弧やパイプ構文を含む曖昧な記法を排除する
  - _Requirements: 2.1, 2.2, 2.3_

- [x] 2. Fixing フェーズのプロンプトをスキルに委譲する
- [x] 2.1 コンフリクト検出・解決のインライン記述とそのフラグパラメータを除去する
  - スキルが自律的に検出・対応するため、マージコンフリクト解決のインライン手順をプロンプトから除外する
  - コンフリクト状態フラグのパラメータをプロンプト生成の設定関数から除去する
  - _Requirements: 1.3, 1.6, 1.7_

- [x] 2.2 デザインフェーズの fixing プロンプトをスキル委譲形式に書き換える
  - `Run /cupola:fix design` を呼び出す簡略化されたプロンプトを生成する
  - コミットメッセージ指示・品質チェック手順・git 操作手順等のインライン記述を削除する
  - PR番号・Issue番号・言語等のコンテキスト情報を引き続き含める
  - output-schema セクション（レビュースレッドへの応答フォーマット）を引き続き含める
  - _Requirements: 1.1, 1.3, 1.4, 1.5, 1.6, 1.7_

- [x] 2.3 実装フェーズの fixing プロンプトをスキル委譲形式に書き換える
  - `Run /cupola:fix impl` を呼び出す簡略化されたプロンプトを生成する
  - 2.2 と同じコンテキスト情報と output-schema を維持する
  - _Requirements: 1.2, 1.3, 1.4, 1.5, 1.6, 1.7_

- [x] 2.4 プロンプト生成の設定からコンフリクトフラグを完全に除去する
  - コンフリクトフラグのパラメータをプロンプト設定の構成箇所からすべて削除する
  - _Requirements: 1.7_

- [x] 3. Fixing プロンプトの呼び出し側を更新する
- [x] 3.1 プロンプト設定の呼び出しからコンフリクトフラグの引数を除去する
  - コンフリクト検出のロジックとワーニングログは維持する
  - プロンプト設定の呼び出しからコンフリクトフラグを削除する
  - _Requirements: 1.7_

- [x] 4. テストを新しいプロンプト仕様に合わせて更新する
- [x] 4.1 削除されたインライン手順に依存するテストを削除する
  - コミットメッセージ指示・品質チェック手順・git 操作手順等のインライン記述がプロンプトに含まれることを検証していたテストを削除する
  - コンフリクトフラグのパラメータを受け取っていたテスト呼び出しを更新する
  - _Requirements: 3.2, 3.3_

- [x] 4.2 スキル委譲を検証する新規テストを追加する
  - デザインフェーズでスキル呼び出し指示がプロンプトに含まれることを検証する
  - 実装フェーズでスキル呼び出し指示がプロンプトに含まれることを検証する
  - 削除されたインライン手順がプロンプトに含まれないことを検証する
  - _Requirements: 3.1, 3.2_

- [x] 4.3 output-schema セクションの検証テストを維持・更新する
  - レビュースレッドがある場合の出力フォーマット（thread_id・response・resolved）を検証する
  - レビュースレッドがない場合の出力フォーマット（`{"threads": []}` ）を検証する
  - _Requirements: 3.4_
