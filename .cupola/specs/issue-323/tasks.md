# 実装タスク一覧

- [ ] 1. 基盤となるコンテキスト構造と共有ヘルパーを導入する
- [ ] 1.1 (P) 全 executor が共有するランタイムリソースを束ねる Context struct を定義する
  - 全実行引数（GitHub クライアント・各種リポジトリ・ランナー・設定など）をひとつの struct に集約できる状態になっている
  - port trait への依存のみを持ち、具体型を含まない Clean Architecture 準拠の構造になっている
  - _Requirements: 2.1_

- [ ] 1.2 (P) 複数の executor から呼ばれる共通ヘルパー関数を専用モジュールに移動する
  - PR 番号取得・フェーズ変換・エラー検索などのヘルパーが専用モジュールに集約されている
  - 各ヘルパーは execute モジュール内からのみアクセス可能な可視性に制限されている
  - _Requirements: 1.1, 1.4_

- [ ] 2. 各 Effect 種別の executor モジュールを作成する（Context に依存）
- [ ] 2.1 (P) コメント投稿系 Effect の実装を専用モジュールに分離する
  - 完了・キャンセル・リトライ上限・CI 修正上限・未信頼の各コメント投稿が独立したモジュールに存在する
  - 各関数が Context struct ひとつを受け取る形になっており、`#[allow(clippy::too_many_arguments)]` が不要になっている
  - _Requirements: 1.1, 1.2, 1.5, 2.3, 2.4_

- [ ] 2.2 (P) ワークツリー操作系 Effect の実装を専用モジュールに分離する
  - クリーンアップと実装ブランチへの切り替えが独立したモジュールに存在する
  - 各関数が Context struct ひとつを受け取る形になっている
  - _Requirements: 1.1, 1.2, 1.5, 2.3, 2.4_

- [ ] 2.3 (P) Issue クローズ Effect の実装を専用モジュールに分離する
  - Issue クローズ処理が独立したモジュールに存在する
  - Context struct を受け取る形になっている
  - _Requirements: 1.1, 1.2, 1.5, 2.3, 2.4_

- [ ] 3. SpawnInit Effect の実装を専用モジュールに分離する
  - init タスクの起動・ハンドル準備・同期 git 操作が独立したモジュールに存在する
  - 既存の init 関連テストが新モジュール内に移動されており、テストで使用するモック型も同モジュール内に配置されている
  - _Requirements: 1.1, 1.2, 1.5, 2.3, 2.4, 5.3_

- [ ] 4. SpawnProcess Effect の実装を専用モジュールに分離する
  - プロセス起動・本文改ざん検知が独立したモジュールに存在する
  - 共通ヘルパーモジュールの関数を呼び出す形になっている
  - 既存のプロセス起動関連テストが新モジュール内に移動されている
  - `#[allow(clippy::too_many_arguments)]` が除去されている
  - _Requirements: 1.1, 1.2, 1.5, 2.3, 2.4, 5.3_

- [ ] 5. Effect の種別ごとに対応する executor へルーティングする dispatcher を作成する
  - 全 Effect バリアントが網羅的に対応する executor に委譲されており、バリアント追加漏れがコンパイル時に検出される
  - dispatcher は execute モジュール内からのみアクセス可能な可視性に制限されている
  - Context ひとつを引数として受け取るため `#[allow(clippy::too_many_arguments)]` が不要
  - _Requirements: 1.1, 3.1, 3.2, 3.3_

- [ ] 6. メインエントリポイントを整備して execute/ ディレクトリを完成させる
  - 単一ファイルから execute/ ディレクトリ構成（メインモジュール + 8 サブモジュール）への移行が完了している
  - `execute_effects` のパブリック API シグネチャが変更されておらず、呼び出し元への影響がゼロである
  - メインモジュールの実装が Context 構築と dispatcher への委譲のみになっている
  - 共有ユーティリティ（DB リトライ・本文ハッシュ・改ざんエラー型・Worktree トレイト）がメインモジュールに集約されている
  - _Requirements: 1.1, 1.3, 1.5, 2.2, 4.1, 4.2, 4.3, 5.4_
