# Implementation Plan

- [x] 1. Issue 行の表示に CI修復回数を含める
- [x] 1.1 Issue 行に CI修復回数ラベルを表示する
  - CI修復回数の上限設定がある場合は `ci-fix: N/MAX`、ない場合は `ci-fix: N` と表示される
  - 通知保留の有無にかかわらず、Issue 行に CI修復回数ラベルが一貫して表示される
  - ラベルは worktree パスの後に表示され、既存の ⚠ 表示は末尾に維持される
  - CI修復回数が 0 の場合でもラベルが省略されず表示される
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5_

- [x] 2. テストを整合させる
- [x] 2.1 既存テストの期待値を新しい表示形式に対応させる
  - 既存の出力文字列検証のうち、Issue 行の表示を確認しているものを見直し、必要に応じて更新する
  - 表示形式の変更によって不整合になる期待値を修正する
  - _Requirements: 2.1_

- [x] 2.2 (P) 上限設定がある場合の表示を検証するテストを追加する
  - CI修復回数が 2、上限が 5 の場合、出力に `ci-fix: 2/5` を含む
  - CI修復回数が 0、上限が 5 の場合、出力に `ci-fix: 0/5` を含む
  - _Requirements: 2.2_

- [x] 2.3 (P) 上限設定がない場合の表示を検証するテストを追加する
  - CI修復回数が 3 の場合、出力に `ci-fix: 3` を含み `/` を含まない
  - _Requirements: 2.3_

- [x] 2.4 (P) 通知保留がある場合の表示を検証するテストを追加する
  - CI修復回数が上限を超えていて通知保留中の場合、出力に `ci-fix: 6/5 ⚠ ci-fix-limit notification pending` を含む
  - _Requirements: 2.4_
