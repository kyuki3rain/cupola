# 実装計画

- [x] 1. 認証に用いる秘密情報を安全に扱えるようにする
  - アプリケーション内で秘密情報を平文のまま保持し続けない取り扱いに移行する
  - 変更後も既存の動作を損なわないことを確認する
  - _Requirements: 1.1, 1.2_

- [x] 2. 各通信経路で認証情報を一貫して安全に扱えるようにする

- [x] 2.1 (P) 外部 API 呼び出しでの認証情報の保持と利用を見直す
  - 認証情報が安全な形で受け渡し・保持されるようにする
  - 認証に利用する箇所でのみ必要最小限の形で値を利用するようにする
  - テスト用の初期化経路でも本番と同じ取り扱い方針を維持する
  - _Requirements: 2.1, 2.3, 2.5_

- [x] 2.2 (P) 別経路の API 呼び出しでも認証情報の取り扱いを統一する
  - 保持する認証情報を安全な形で扱うようにする
  - 初期化から利用まで同じ前提で受け渡されるようにそろえる
  - 認証文字列が必要な箇所では、意図が明確な形でのみ利用されるようにする
  - これにより、通常の保持・受け渡しでは平文文字列を避けつつ、必要時のみアクセスする
  - _Requirements: 2.2, 2.4, 2.5_

- [x] 3. 起動時から利用時まで認証情報の受け渡しを安全な形に統一する
  - 起動時に取得した認証情報を以後の処理で安全に扱う
  - 各起動経路から利用先まで一貫した形で認証情報が渡されるようにする
  - 変更後も既存の起動フローとクライアント生成の挙動を維持する
  - _Requirements: 3.1, 3.2_

- [x] 4. 秘密情報の露出防止を検証し品質確認する

- [x] 4.1 秘密情報が出力に露出しないことを検証する
  - デバッグ出力や失敗時出力に実際の認証情報が含まれないことを確認する
  - ログや診断情報から秘密情報が露出しないことを担保する
  - _Requirements: 4.1, 4.2_

- [x] 4.2 品質保証を確認する
  - 品質チェックを実行し、警告を含めて品質ゲートを通過することを確認する
  - 関連するテストを実行し、既存機能を含めて成功することを確認する
  - _Requirements: 5.1, 5.2, 5.3_
