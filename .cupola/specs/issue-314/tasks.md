# 実装タスク

## タスク概要

panic 時の graceful exit 実装を 4 つの主要タスクに分割する。タスク 1 が基盤（ChildProcessRegistry）を提供し、タスク 2・3 は基盤の上に並行して実装できる。タスク 4（テスト）は全実装完了後に実施する。

---

- [ ] 1. ChildProcessRegistry の実装と SessionManager への統合

- [ ] 1.1 ChildProcessRegistry 型を実装する
  - application 層に子プロセスの PID を管理する共有レジストリ型を新たに実装する
  - スレッド境界を越えた安全な並行アクセスができるよう設計し、複数の所有者間で安価にコピーできること
  - 子プロセス PID の登録メソッドと登録解除メソッドを実装する
  - 同期コンテキストから全登録プロセスに SIGTERM を送信し、タイムアウト後に SIGKILL を送るシャットダウンメソッドを実装する（async/await 不使用）
  - Mutex が毒化していても内部値を回収してシャットダウンを続行できるようにする
  - 全クリーンアップ操作はエラーを無視するベストエフォートで実装する
  - PID の登録・登録解除が正しく動作することをユニットテストで確認する
  - _Requirements: 1.1, 1.4, 2.1, 2.2, 2.3, 2.4, 2.5_

- [ ] 1.2 SessionManager に ChildProcessRegistry を統合する
  - `SessionManager` にレジストリをオプション設定するビルダーメソッドを追加する
  - 子プロセスが起動したとき、レジストリに PID を登録する
  - 子プロセスが終了・回収されたとき、レジストリから PID を削除する
  - 子プロセスが強制終了されたとき（単体・一括問わず）、レジストリから PID を削除する
  - レジストリが設定されていない場合は既存の動作を維持する（後方互換性）
  - レジストリなしの場合に既存動作が変わらないことをユニットテストで確認する
  - レジストリありの場合に子プロセスのライフサイクルに応じて PID が正しく登録・削除されることをユニットテストで確認する
  - _Requirements: 1.2, 1.3, 1.5_

- [ ] 2. (P) PollingUseCase への registry 転送を実装する
  - `PollingUseCase` に既存のビルダーメソッドと同じパターンで、レジストリを `SessionManager` に転送するビルダーメソッドを追加する
  - タスク 1 完了後に着手可能
  - _Requirements: 4.3_

- [ ] 3. (P) install_panic_hook を拡張する
  - `install_panic_hook` がレジストリを受け取れるようシグネチャを拡張する
  - パニック時の処理順序を「子プロセスシャットダウン → PID ファイル削除 → ログ記録 → 前のフック呼び出し」に更新する
  - フォアグラウンドとデーモン子プロセスの両起動パスでレジストリを作成し、panic hook と PollingUseCase の両方に渡す
  - タスク 1 完了後に着手可能（タスク 2 と並行実施可）
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 3.5, 4.1, 4.2, 4.4_

- [ ] 4. テストを実装する

- [ ] 4.1 (P) ChildProcessRegistry と SessionManager のユニットテスト（拡張）
  - 実際のプロセスを起動してレジストリに登録し、シャットダウン API 呼び出し後にプロセスが終了していることを確認する
  - タイムアウト後に SIGKILL が送信されて残存プロセスが終了することを確認する
  - 空のレジストリに対してシャットダウン API を呼び出しても安全であることを確認する
  - _Requirements: 5.5_

- [ ] 4.2 (P) panic hook の統合テスト
  - テストプロセスを起動してレジストリに登録し、panic hook を直接呼び出した後にプロセスが消えていることを確認する
  - panic hook 実行後に PID ファイルが削除されていることを確認する
  - テスト全体が SIGTERM タイムアウト＋マージン以内に完了することを確認する
  - Mutex 毒化状態でも子プロセスが正常に回収されることを確認する
  - _Requirements: 5.1, 5.2, 5.3, 5.4_
