# Implementation Plan

- [x] 1. spec-init スキルの提供
- [x] 1.1 新しい spec を初期化できるスキルを提供する
  - 既存スキル群と一貫した利用体験で呼び出せるようにする
  - スキルの目的と利用方法が利用者に分かりやすく伝わるようにする
  - 初期化に必要な操作を実行できるようにする
  - _Requirements: 1.1, 1.3_

- [x] 1.2 入力と前提条件を検証し、失敗時に案内を返せるようにする
  - 必須入力が不足している場合は、実行せずに使い方を案内する
  - 利用できない識別子が指定された場合は、理由を明示してエラーを返す
  - 初期化に必要な前提が満たされていない場合は、次に取るべき行動を案内する
  - 対象がすでに存在する場合は、安全にスキップして利用者へ通知する
  - _Requirements: 1.2, 4.1, 4.2, 4.3_

- [x] 1.3 言語を適切に決定して初期化内容へ反映できるようにする
  - 利用者が明示した言語指定を優先して使用する
  - 明示指定がない場合は既存設定から言語を解決する
  - 利用可能な設定がない場合は既定の言語で初期化する
  - _Requirements: 1.3, 1.4_

- [x] 1.4 spec の初期定義を生成できるようにする
  - 新しい spec に必要な基本情報を含む初期定義を用意する
  - spec 名、生成時刻、言語などの初期値が正しく反映されるようにする
  - 既定の雛形が利用できない場合でも初期化を継続できるようにする
  - 初期化した spec の作業を開始できる状態まで成果物を作成する
  - _Requirements: 2.2, 2.4, 3.1, 3.2, 3.3, 3.4, 3.5, 3.6_

- [x] 1.5 requirements の初期内容を用意し、次の作業を案内する
  - 要件整理を開始できる初期コンテンツを生成する
  - 既定の雛形が利用できない場合でも必要最小限の内容を提供する
  - 初期化で作成された成果物が利用者に分かるように伝える
  - 初期化完了後に次の推奨ステップを案内する
  - _Requirements: 2.1, 2.3, 2.5, 3.7, 1.5_

- [x] 2. セットアップ後に spec-init スキルを利用可能にする
- [x] 2.1 既存のスキル提供フローに沿って spec-init を利用可能にする
  - 既存スキルと同様の提供方法で新スキルを追加する
  - 既存の動作を損なわずに新スキルを利用可能にする
  - _Requirements: 5.1, 5.2, 5.3_

- [x] 2.2 ビルド・テスト・静的検証を通じて品質を確認する (P)
  - 変更後もビルドが成功することを確認する
  - 既存テストが継続して成功することを確認する
  - 追加した変更に重大な警告が残らないことを確認する
