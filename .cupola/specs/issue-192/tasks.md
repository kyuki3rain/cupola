# 実装計画

- [ ] 1. ロケールキーを追加して各言語でコメント案内を提供できるようにする
- [ ] 1.1 (P) 英語環境で CI 修正上限に達したことを案内できるようにする
  - 利用者に対して、CI 修正の上限に達したことが英語で伝わる文面を用意する
  - 案内文には上限回数の値を自然に埋め込めるようにする
  - _Requirements: 1.1_

- [ ] 1.2 (P) 日本語環境で CI 修正上限に達したことを案内できるようにする
  - 利用者に対して、CI 修正の上限に達したことが日本語で伝わる文面を用意する
  - 案内文には上限回数の値を自然に埋め込めるようにする
  - _Requirements: 1.2_

- [ ] 1.3 英語環境で未信頼ユーザーの操作を拒否したことを案内できるようにする
  - 利用者に対して、操作を受け付けられない理由が英語で明確に伝わる文面を用意する
  - 1.1 と同一ファイルを更新するため、1.1 完了後に着手する
  - _Requirements: 2.1_

- [ ] 1.4 日本語環境で未信頼ユーザーの操作を拒否したことを案内できるようにする
  - 利用者に対して、操作を受け付けられない理由が日本語で明確に伝わる文面を用意する
  - 1.2 と同一ファイルを更新するため、1.2 完了後に着手する
  - _Requirements: 2.2_

- [ ] 2. コメント投稿処理を利用者の言語に応じた案内と適切な失敗記録に対応させる
- [ ] 2.1 CI 修正上限到達時の案内を利用者の言語に応じて投稿できるようにする
  - 固定文ではなく、利用者の言語設定に応じた案内文が使われるようにする
  - 案内文に CI 修正上限の値が正しく反映されるようにする
  - タスク 1 完了後に着手する
  - _Requirements: 1.3, 1.4, 1.5, 1.6_

- [ ] 2.2 未信頼ユーザーへの案内を利用者の言語に応じて投稿し、失敗時は警告を残すようにする
  - 固定文ではなく、利用者の言語設定に応じた案内文が使われるようにする
  - コメント投稿に失敗した場合、失敗を握り潰さず警告として記録されるようにする
  - タスク 1 完了後に着手する
  - _Requirements: 2.3, 3.1, 3.2, 3.3_

- [ ] 3. ユニットテストを追加する
- [ ] 3.1 (P) CI 修正上限到達時の案内が言語設定に応じて変わることをテストする
  - 日本語設定時に日本語の案内が投稿されることを検証する
  - 案内文に上限回数の値が正しく反映されることを検証する
  - タスク 2.1 完了後に着手する
  - _Requirements: 4.2, 4.4_

- [ ] 3.2 (P) 未信頼ユーザーへの案内でコメント投稿失敗時に警告が記録されることをテストする
  - コメント投稿が失敗した場合に失敗が無視されないことを検証する
  - モックで投稿失敗を発生させたときに警告が記録されることを確認する
  - タスク 2.2 完了後に着手する
  - _Requirements: 4.3_

- [ ] 4. 品質チェックを実施する
  - `devbox run clippy` を実行してエラーがないことを確認
  - `devbox run test` を実行して全テストがパスすることを確認
  - `devbox run fmt-check` を実行してフォーマットが正しいことを確認
  - _Requirements: 4.1_
