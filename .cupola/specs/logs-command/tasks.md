# Implementation Plan

## Task Format

- `(P)` = 並行実行可能なタスク
- `- [ ]*` = 任意（MVP 後に対応可能）なテストタスク

---

- [ ] 1. `logs` サブコマンドの CLI 定義を追加する

- [ ] 1.1 (P) `Command` enum に `Logs` バリアントを定義する
  - `-f` / `--follow` フラグ（`bool`、デフォルト: false）を追加する
  - `--config` オプション（`PathBuf`、デフォルト: `.cupola/cupola.toml`）を追加する
  - コマンドの説明文（`--help` 表示用）を適切に記述する
  - 追加後、既存の `match cli.command` でコンパイルエラーが出ることを確認する（網羅性チェック）
  - _Requirements: 4.1, 4.2, 4.3_

- [ ] 2. ログファイル発見とエラーハンドリングを実装する

- [ ] 2.1 (P) ログディレクトリから最新ログファイルを発見する機能を実装する
  - `bootstrap/logs_command.rs` を新規作成する
  - `log_dir` 内のエントリを列挙し、`cupola.` プレフィックスを持つファイルのみを対象とする
  - 辞書順ソートにより最新ファイル（ISO 8601 日付サフィックス）を選択する
  - `log_dir` が存在しない場合、ファイルが存在しない場合にそれぞれ適切なエラーを返す
  - _Requirements: 1.2, 3.3, 3.4_

- [ ] 2.2 (P) 設定ミスによるエラーをハンドリングする機能を実装する
  - `log_dir` が `None`（`log.dir` 未設定）の場合に `"log.dir が cupola.toml に設定されていません"` を stderr に出力して非ゼロ終了する
  - ディレクトリが存在しない場合に `"ログディレクトリが存在しません: {path}"` を stderr に出力して非ゼロ終了する
  - ログファイルが存在しない場合に `"ログファイルが見つかりません: {dir}"` を stderr に出力して非ゼロ終了する
  - _Requirements: 3.1, 3.2, 3.3, 3.4_

- [ ] 3. 通常ログ表示モードを実装する

- [ ] 3.1 ファイル末尾20行の読み取りと標準出力への表示を実装する
  - ファイル全行を読み取り、末尾20行を取得する（20行未満の場合は全行）
  - 取得した行を stdout に順番に出力する
  - 内容を変換・加工せずそのまま出力する（改行を維持する）
  - _Requirements: 1.1, 1.3_

- [ ] 4. リアルタイム追跡モードを実装する

- [ ] 4.1 ポーリングベースのリアルタイムログ追跡を実装する
  - 起動時に末尾20行を初期表示する
  - ファイルの末尾バイトオフセットを記録し、200ms ごとに新規バイトを読み取る
  - 新規書き込みがある場合は即座に stdout へ出力する
  - `-f` フラグが指定された場合のみこのループを実行する
  - _Requirements: 2.1, 2.2_

- [ ] 4.2 Ctrl+C によるリアルタイム追跡の安全終了を実装する
  - `tokio::signal::ctrl_c()` で SIGINT を非同期に捕捉する
  - ポーリングループとシグナル待ちを `tokio::select!` で競合させる
  - SIGINT 受信時に正常終了コード（0）でプロセスを終了する
  - _Requirements: 2.3_

- [ ] 5. Bootstrap 層に統合して動作を結線する

- [ ] 5.1 `app.rs` に `Command::Logs` のルーティングと初期化を追加する
  - `match cli.command` の `Command::Logs` ブランチで `load_toml(config)` を呼び出す
  - `CupolaToml` から `log.dir` を抽出して `run_logs(log_dir, follow)` に渡す
  - `bootstrap/mod.rs` に `pub mod logs_command;` を追加する
  - _Requirements: 3.1, 4.1_

- [ ] 6. テストを実装する

- [ ] 6.1 (P) ログファイル発見ロジックのユニットテストを実装する
  - 複数ファイルが存在する場合に最新ファイルを正しく選択することを検証する
  - `cupola.` プレフィックスを持たないファイルを除外することを検証する
  - ディレクトリが空の場合にエラーを返すことを検証する
  - _Requirements: 1.2, 3.3, 3.4_

- [ ] 6.2 (P) 末尾行取得ロジックのユニットテストを実装する
  - 20行未満のファイルに対してすべての行を返すことを検証する
  - 20行超のファイルに対して末尾20行のみを返すことを検証する
  - _Requirements: 1.1, 1.3_

- [ ]* 6.3 通常モードとエラーハンドリングの統合テストを実装する
  - 一時ディレクトリにダミーログファイルを作成し、`run_logs(follow=false)` が末尾20行を返すことを検証する
  - `log_dir=None` 時に適切なエラーを返すことを検証する
  - 空ディレクトリ時に適切なエラーを返すことを検証する
  - _Requirements: 1.1, 1.2, 1.3, 3.2, 3.3, 3.4_
